Prerequisites

Before installing HoxCore, ensure you have the following:

  • Python 3.8 or higher
  • pip — Python package installer
  • Git — required for version control features

Check your Python version

python --version
# Python 3.8+ required

Installation

Standard installation

pip install hoxcore

Install from source

git clone https://github.com/sdescobedo/hoxcore.git
cd hoxcore
pip install -e .

Development dependencies

pip install hoxcore[dev]

Virtual environment recommended

# Create and activate
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate

pip install hoxcore

Verify Installation

hxc --version
hxc --help

You should see:

hxc version 0.1.0

Installation successful

If the version number appears, HoxCore is ready to use.

Initialize a Registry

A registry is a dedicated Git repository containing only YAML metadata. Think of it as a Git repository for your project definitions.

# Initialize in a new directory
hxc init my-registry

# Or in the current directory
hxc init .

Every registry is a Git repository

This gives you version history, branching, remote backup, and full auditability of all changes—automatically.

Registry Structure

my-registry/
├── .git/           # Git repository
├── .hxc/           # HoxCore internal data
├── config.yml      # Registry configuration
├── programs/       # Program definitions
├── projects/       # Project definitions
├── missions/       # Mission definitions
├── actions/        # Action definitions
└── templates/      # Custom templates
DirectoryEntity TypePurpose
programs/ProgramStrategic containers grouping related initiatives
projects/ProjectConcrete efforts with defined outputs
missions/MissionEvent-driven efforts with clear culmination
actions/ActionOngoing activities without a defined end
templates/TemplateCustom scaffolding templates

Create a Project

# Minimal
hxc create project --title "My First Project"

# With details
hxc create project \
  --title "AI-Powered CLI Tool" \
  --description "Building an intelligent command-line interface" \
  --category software.dev/cli-tool \
  --id P-001 \
  --tags cli,ai,python \
  --due-date 2024-12-31

# Interactive mode
hxc create project --interactive

HoxCore automatically assigns an immutable UID (e.g. proj_abc123def456), sets the status to active, and creates a YAML file in projects/.

List Entities

hxc list projects
hxc list programs
hxc list all

# Filter
hxc list projects --status active
hxc list projects --category software.dev
hxc list projects --tags ai,cli
UID                    ID      Title                 Status    Category
proj_abc123def456      P-001   My First Project      active    software.dev/cli-tool
proj_xyz789ghi012      P-002   Backend API           active    software.dev/api

View Entity Details

# By UID or ID
hxc show proj_abc123def456
hxc show P-001

# Show raw YAML
hxc show P-001 --raw

# Show Git history
hxc show P-001 --history

Update an Entity

# Update status
hxc update P-001 --status completed

# Multiple fields
hxc update P-001 \
  --status completed \
  --completion-date 2024-06-30 \
  --add-tags finished,deployed

# Tag management
hxc update P-001 --add-tags production
hxc update P-001 --remove-tags experimental

# Open in $EDITOR
hxc update P-001 --edit

UIDs are immutable

The uid field cannot be changed after creation. This guarantees referential integrity across all relationships.

Query Entities

# Full-text search
hxc query --search "machine learning"

# Tag filters (AND logic)
hxc query --tags ai,python

# Tag filters (OR logic)
hxc query --any-tags ai,ml,data

# Date range
hxc query projects \
  --status active \
  --due-before 2024-12-31

# Overdue projects
hxc query projects \
  --status active \
  --due-before $(date +%Y-%m-%d)

Set Registry Path

# Set default registry
hxc registry path --set /path/to/my-registry

# View current registry
hxc registry path

# Override per-command
hxc list projects --registry /path/to/other-registry

Registry discovery order

  1. Current directory
  2. Parent directories (up to filesystem root)
  3. ~/.hxc/registry
