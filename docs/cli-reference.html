Global Options

These options apply to every hxc command:

FlagDescription
--registry PATHOverride registry path for this command
--output json|yaml|tableOutput format (default: table)
--no-colorDisable colored output
--quiet, -qSuppress non-essential output
--verbose, -vEnable verbose output
--versionPrint version and exit
--help, -hShow help for command

hxc init

Initialize a new HoxCore registry in the specified directory.

hxc init [PATH] [OPTIONS]

Arguments

ArgumentDescription
PATHDirectory to initialize (default: current directory)

Options

FlagDescription
--name NAMERegistry name (used in config.yml)
--no-gitSkip Git initialization
--template NAMEBootstrap from a registry template

Examples

hxc init my-registry
hxc init . --name "Personal Registry"
hxc init work-projects --no-git

hxc create

Create a new entity in the registry.

hxc create {program|project|mission|action} [OPTIONS]

Options

FlagDescription
--title TEXTEntity title (required unless --interactive)
--id TEXTHuman-readable ID (e.g. P-001). Auto-generated if omitted.
--description TEXTDescription text
--status STATUSInitial status (default: active)
--category TEXTCategory string
--tags TEXTComma-separated tags
--parent UIDParent entity UID
--start-date DATEStart date (YYYY-MM-DD)
--due-date DATEDue date (projects only)
--template NAMEBootstrap from a named template
--interactive, -iInteractive prompt mode
--editOpen in $EDITOR after creation

Examples

# Minimal project
hxc create project --title "New API"

# Fully specified
hxc create project \
  --title "GraphQL Migration" \
  --id P-042 \
  --category software.dev/api \
  --tags graphql,migration,backend \
  --parent prog_abc123 \
  --due-date 2024-12-31

# Interactive mode
hxc create project --interactive

# From template
hxc create project --template software-project

hxc list

List entities in the registry.

hxc list {programs|projects|missions|actions|all} [OPTIONS]

Options

FlagDescription
--status STATUSFilter by status
--tags TEXTFilter: all listed tags must be present (AND)
--any-tags TEXTFilter: any listed tag present (OR)
--category TEXTFilter by category prefix
--parent UIDFilter by parent entity
--sort FIELDSort by field: title, status, created, updated, due
--descReverse sort order
--limit NMaximum results to return
--output json|yaml|tableOutput format

Examples

hxc list projects
hxc list all --status active
hxc list projects --tags ai,python --sort due
hxc list projects --category software.dev --output json

hxc show

Display full details of a single entity.

hxc show {UID|ID} [OPTIONS]

Options

FlagDescription
--rawPrint raw YAML file contents
--historyShow Git commit history for this entity
--diff REFShow diff against a Git ref (commit, branch, tag)

Examples

hxc show P-001
hxc show proj_abc123def456
hxc show P-001 --raw
hxc show P-001 --history
hxc show P-001 --diff HEAD~3

hxc update

Update fields on an existing entity.

hxc update {UID|ID} [OPTIONS]

Options

FlagDescription
--title TEXTNew title
--description TEXTNew description
--status STATUSNew status
--category TEXTNew category
--add-tags TEXTComma-separated tags to add
--remove-tags TEXTComma-separated tags to remove
--set-tags TEXTReplace all tags with this list
--parent UIDSet parent entity UID
--due-date DATESet due date
--completion-date DATESet completion date
--editOpen in $EDITOR

Examples

hxc update P-001 --status completed
hxc update P-001 --add-tags production,deployed
hxc update P-001 --edit
hxc update P-001 --status completed --completion-date 2024-06-30

hxc delete

Delete an entity from the registry. The file is removed and the deletion is committed to Git.

hxc delete {UID|ID} [OPTIONS]

Prefer archiving over deletion

Deleting an entity removes its UID from the registry, which may break references in other entities. Consider hxc update P-001 --status archived instead to preserve history and referential integrity.

Options

FlagDescription
--force, -fSkip confirmation prompt
--message TEXTCustom Git commit message

Examples

hxc delete P-001
hxc delete proj_abc123 --force

hxc query

Advanced search and filtering across all entities.

hxc query [TYPE] [OPTIONS]

Options

FlagDescription
--search TEXTFull-text search across title, description, tags
--tags TEXTAll tags must match (AND)
--any-tags TEXTAny tag must match (OR)
--status STATUSStatus filter
--category TEXTCategory prefix filter
--due-before DATEDue date before (YYYY-MM-DD)
--due-after DATEDue date after (YYYY-MM-DD)
--created-before DATECreated before date
--created-after DATECreated after date
--has-tag TEXTEntity has this specific tag
--parent UIDEntities with this parent

Examples

# Find all active AI projects
hxc query projects --tags ai --status active

# Find overdue items
hxc query --status active --due-before $(date +%Y-%m-%d)

# Full-text search
hxc query --search "machine learning"

# Combine filters
hxc query projects \
  --status active \
  --category software.dev \
  --due-before 2024-12-31 \
  --output json

hxc template

Manage and apply entity templates.

hxc template {list|show|apply|create} [OPTIONS]

Subcommands

SubcommandDescription
hxc template listList available templates
hxc template show NAMEDisplay template definition
hxc template apply NAMEApply template to create entity
hxc template create --from UIDCreate template from existing entity

Examples

hxc template list
hxc template show software-project
hxc template apply software-project --title "New Service"
hxc template create --from P-001 --name my-project-template

hxc registry

Manage registry configuration and paths.

hxc registry {path|config|validate} [OPTIONS]

Subcommands

SubcommandDescription
hxc registry pathShow current registry path
hxc registry path --set PATHSet default registry path in ~/.hxc/config
hxc registry configShow registry config.yml
hxc registry validateValidate all entity files

hxc validate

Validate entity YAML files against the schema.

hxc validate [PATH] [OPTIONS]

Options

FlagDescription
--strictTreat warnings as errors
--fixAuto-fix safe issues (e.g. missing updated_at)
 
hxc validate
hxc validate projects/P-001.yaml
hxc validate --strict

hxc export

Export registry data to various formats.

hxc export [TYPE] [OPTIONS]

Options

FlagDescription
--format json|csv|markdownExport format
--output FILEWrite to file (default: stdout)
--status STATUSFilter by status before export
--tags TEXTFilter by tags before export
 
hxc export projects --format json --output projects.json
hxc export all --format csv --output registry.csv
hxc export projects --format markdown --status active
